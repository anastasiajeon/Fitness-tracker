# Фитнес-трекер с графиками
Веб-приложение для отслеживания фитнес-тренировок с визуализацией прогресса и детальной статистикой.

# Возможности
Дашборд с графиками - Визуализация прогресса по калориям и типам тренировок

Добавление тренировок - Интуитивная форма ввода с предпросмотром

Детальная статистика - Фильтрация по периодам и типам тренировок

История тренировок - Поиск, просмотр и управление всеми записями

Адаптивный дизайн - Оптимизирован для всех устройств

Локальное хранение - Данные сохраняются в браузере (localStorage)

# Структура проекта

# Описание функций в модуле DATA-MANAGER.JS

## getWorkouts() - Получение всех тренировок
Назначение: Чтение данных из localStorage с обработкой ошибок

Почему выбрана:

- Использует try-catch для защиты от битых данных

- Возвращает пустой массив по умолчанию (fail-safe подход)

- Логирует ошибки для отладки

## saveWorkouts(workouts) - Сохранение всех тренировок
Назначение: Запись данных в localStorage

Почему выбрана:

- Обработка ошибок записи

- Логирование для отслеживания операций

- Использует JSON.stringify для сериализации объектов

## addWorkout(workout) - Добавление новой тренировки
Назначение: Добавление новой тренировки в коллекцию

Почему выбрана:

- Генерирует уникальный ID через Date.now() + Math.random()

- Объединяет чтение, модификацию и сохранение в одной функции

- Возвращает добавленный объект для возможного дальнейшего использования

## deleteWorkout(id) - Удаление тренировки по ID
Назначение: Удаление конкретной тренировки

Почему выбрана:

- Использует filter() для иммутабельного удаления

- Возвращает boolean результат для обратной связи

- Сравнивает длины массивов для определения успешности операции

## filterWorkouts({period, type, search}) - Фильтрация тренировок
Назначение: Многофакторная фильтрация тренировок

Почему выбрана:

- Использует деструктуризацию параметров с значениями по умолчанию

* Поддерживает три типа фильтрации: по периоду, типу и текстовому поиску

- Возвращает отсортированный по дате результат

## getStatistics(workouts) - Расчет статистики
Назначение: Агрегация и расчет статистических данных

Почему выбрана:

- Использует reduce() для суммирования значений

- Рассчитывает средние значения

- Группирует данные по дням и типам тренировок для графиков

## getTypeName(typeKey) и getIntensityName(intensityKey)
Назначение: Преобразование ключей в читаемые названия

Почему выбраны:

- Используют объекты как словари для O(1) доступа

- Возвращают значения по умолчанию при отсутствии ключа

- Отделяют данные от логики отображения

# Описание функций в модуле CHARTS.JS

## initCaloriesChart(data) - График калорий
Назначение: Создание линейного графика прогресса калорий

Почему выбрана:

- Линейный график идеален для отображения прогресса во времени

- Настройки Chart.js обеспечивают адаптивность и интерактивность

- Включает tooltips для детальной информации

## initWorkoutTypesChart(data) - Круговая диаграмма типов
Назначение: Визуализация распределения тренировок по типам

Почему выбрана:

- Doughnut-диаграмма лучше показывает пропорции, чем pie

- Легенда справа улучшает читаемость

- Процентные значения в tooltips

## updateDashboardCharts() - Обновление графиков дашборда
Назначение: Координация обновления всех графиков на дашборде

Почему выбрана:

- Централизует логику обновления графиков

- Переиспользует существующие функции инициализации

- Обеспечивает согласованность данных

# Описание функций в модуле SCRIPT.JS (ОСНОВНЫЕ ФУНКЦИИ)

## showNotification(message, type) - Уведомления
Назначение: Показ всплывающих уведомлений

Почему выбрана:

- Динамическое создание DOM-элементов

- Поддержка разных типов (success, error, info)

- Автоматическое скрытие через таймер

- CSS-анимации для плавного появления

## initNavigation() - Управление навигацией
Назначение: Обработка переключения между вкладками

Почему выбрана:

- Использует делегирование событий

- Поддерживает историю браузера (pushState)

- Обеспечивает плавную прокрутку к якорям

- Обрабатывает кнопку логотипа

## saveWorkout() - Сохранение тренировки (форма)
Назначение: Обработка отправки формы добавления тренировки

Почему выбрана:

- Полная валидация всех полей

- Постепенная проверка с фокусировкой на проблемных полях

- Интеграция с DataManager

- Пользовательские уведомления о результате

## loadHistory(search) - Загрузка истории тренировок
Назначение: Отображение списка тренировок с поиском

Почему выбрана:

- Использует шаблонные строки для генерации HTML

- Обрабатывает empty state (состояние без данных)

- Динамически добавляет обработчики событий

- Поддерживает поиск по заметкам и типам

## initModal() - Управление модальным окном
Назначение: Управление модальным окном подтверждения удаления

Почему выбрана:

- Использует замыкание для хранения workoutToDelete

- Обрабатывает клавишу Escape

- Закрытие по клику вне модального окна


- Правильная фокусировка для доступности
